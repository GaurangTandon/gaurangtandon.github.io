About CSSunflower

A pure HTML+CSS game by Gaurang Tandon.

How to play?

Guide sunlight from the sun ☀ to the plant 🌿 by rotating mirrors.

Click a mirror to rotate it 90°. Find the arrangement that sends the beam all the way to the plant.

Use Easy, Hard, or Random to generate a new puzzle, or configure a custom grid with the ⚙ button.

Hit Solution to reveal the answer if you're stuck.

What's special about this?

The game UI and logic is implemented in HTML+CSS only (no JavaScript).

Note: The actual game is in an iframe which does not have JavaScript. The root frame uses JavaScript to generate the actual game, add sound effects and other interactive features.

What is the logic to find valid paths here?
  1. Build a directed graph where each sun contributes one node, each plant contributes one node, and each mirror contributes four nodes (one for each orientation)
  2. Connect node A to node B if light can move from A to B. Note that A cannot be plant and B cannot be a sun.
  3. Start a depth-first traversal from the sun and enumerate all possible paths - even visiting previously visited nodes. There are no cycles here.
  4. For each path, once it reaches a dead-end, move to the next part below..
How is the lighting of the path implemented in pure CSS?
  1. Each node in the graph has a corresponding ID. Sun #sun-x-y. Plant: #plant-x-y. Triangle: #t-x-y-s0, #t-x-y-s1, #t-x-y-s2 and #t-x-y-s3
  2. Each cell in the grid also has four light beams #l-x-y-0/1/2/3 for each of the four directions light can flow in.
  3. For example: the path is: Sun(3,5) -> Triangle(3,6,2) -> Triangle()
  4. Note: The actual output of the selectors can be optimized. That would be a good for a v2.
Can I make my own puzzles?

Yes! Click the ⚙ button to get started.

Did you use AI to implement this?

I planned on pen-and-paper how I wanted each part of game to work (sun / plant / mirror / lights and their CSS selectors). I told this to Opus 4.6 on high thinking effort and gave it Playwright access so it can implement on its own. The code is too verbose for my liking, but it works. I still had to manually step in a few times to correct the directions.
